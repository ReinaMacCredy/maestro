# Method Selection

## Algorithm

1. **Assess Context**
   - Problem type (open-ended, constrained, technical, creative)
   - Time available
   - Participant expertise
   - Desired outcome

2. **Match to Technique**

| Context | Recommended Methods |
|---------|-------------------|
| New product ideas | SCAMPER, Reverse Brainstorming, Six Thinking Hats |
| Problem solving | 5 Whys, Fishbone Diagram, Mind Mapping |
| Feature ideation | Brainwriting, Crazy 8s, How Might We |
| Process improvement | SWOT, Force Field Analysis, Starbursting |
| Creative blocks | Random Word, Attribute Listing, Analogy |

3. **Quick Selection Guide**

```
IF problem is well-defined → Structured methods (5 Whys, SCAMPER)
IF problem is ambiguous → Exploratory methods (Mind Mapping, Free Association)
IF need many ideas quickly → Rapid methods (Crazy 8s, Brainwriting)
IF need deep analysis → Analytical methods (Six Thinking Hats, SWOT)
```

## Output Format

```markdown
**Selected Method:** [Method Name]
**Reason:** [Why this method fits the context]
**Process:**
1. [Step 1]
2. [Step 2]
3. [Step 3]
**Time Estimate:** [Duration]
```

## Fallback
If unsure, default to **Mind Mapping** - versatile and always applicable.
