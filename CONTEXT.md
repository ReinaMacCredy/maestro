# Maestro

The local-first harness OS for agent-generated codebases. Substrate that LLM coding agents (Claude Code, Codex, Cursor) operate against. Humans steer, agents execute, maestro is the substrate.

## Language

### Knowledge primitives (directories the agent reads)

**design-docs**:
Strategic / architectural documents that capture vision and core beliefs. Human-written, agent-read, with one carved-out exception: the `design-docs/learnings/` subdirectory is agent-writable so verify-FAIL evidence and migrated v1 memory entries can land as durable learnings (ADR-0015). The rest of `design-docs/` stays human-owned.
_Avoid_: brainstorm, notes

**exec-plans**:
First-class decomposition artifacts for multi-PR work, with progress and decision logs checked in. Live in `docs/exec-plans/active/` until done, then move to `completed/`.
_Avoid_: mission, plan-doc

**product-specs**:
Feature specifications with explicit acceptance criteria and non-goals. Agent reads and implements against.
_Avoid_: spec, feature-doc

**references**:
LLM-targeted condensations of upstream library docs (e.g. `opentui-llms.txt`). Stable, agent-read.
_Avoid_: external-docs

**generated**:
Auto-generated reference docs (e.g. wire-contract snapshots, schema dumps). Regenerated by tooling.
_Avoid_: derived, schema

**architecture**:
Mechanically-enforced layering and dependency rules. Lints are the source of truth, not prose.
_Avoid_: layering, structure

**quality-score**:
Per-domain grade tracking gaps over time. Updated nightly by `gc grade`.
_Avoid_: quality, health

**principles**:
Named golden rules ("prefer shared utils", "no YOLO data probing") with a scan command and a fix recipe. Enforced by `gc slop-cleanup`.
_Avoid_: rules, golden-rules

### Execution primitives (the agent's verb-shaped surface)

**worktree**:
Isolated execution environment per task. Per-worktree `.maestro/runs/`, scoped telemetry, scoped logs.
_Avoid_: branch, sandbox

**loop**:
The default execution mode after a task is claimed. Try -> verify -> evidence -> iterate until PASS verdict or stuck-threshold (Ralph Wiggum Loop).
_Avoid_: ralph-loop, agent-loop

**task**:
A unit of PR-shaped work with a lifecycle (states + gated transitions). Can be standalone (light path, no spec) or inside an exec-plan (heavy path).
_Avoid_: ticket, story, work-item

**handoff**:
Artifact emitted at cross-session state transitions, carrying context for the next agent session.
_Avoid_: context-dump, session-state

**grill**:
The interview protocol `maestro-design` and `maestro-plan` skills run (ADR-0016): one question at a time, recommended answer per question, challenges language against CONTEXT.md, updates the glossary inline as terms resolve, offers ADRs sparingly. No dedicated verb; entering the design or plan skill runs the grill.
_Avoid_: interview, Q&A, drill

## Operating modes

**Light path**: task -> worktree -> loop -> handoff. Used for one-off small specs and single-PR work. No exec-plan required.

**Heavy path**: product-spec -> exec-plan -> N tasks (each running the light path internally) -> quality-score update. Used for features and multi-PR work.

Mode is chosen by the agent at task start, based on the request shape.

## Flagged ambiguities

- Earlier maestro vocabulary: `mission` (now `exec-plan`), `spec` (now `product-spec`), `intake` / `brainstorm` (folded into `design-docs` reading and `product-spec` authoring), `session` / `notes` (folded into `handoff`).
- Spec files live at `.maestro/specs/<slug>.md` (maestro convention) not `docs/product-specs/` (article convention); see ADR-0010.
- Pending: default observability adapter scope (Vector + Victoria* vs vendor-neutral vs minimal). Deferred during grill; decide before Phase 3.
