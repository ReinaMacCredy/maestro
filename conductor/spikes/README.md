# Spike Storage

This directory stores spike code for HIGH risk validations during planning.

## Structure

```
conductor/spikes/<track-id>/
├── spike-001/
│   ├── README.md      # Question, criteria, result
│   └── ...            # Throwaway code
└── spike-002/
    └── ...
```

## Lifecycle

1. Created during `pl` Phase 3 (Verification)
2. Referenced in beads for implementation guidance
3. Archived or deleted after track completion
